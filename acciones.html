Acciones

Todo el funcionamiento de un proyecto se programa mediante bloques de Código. Tanto los objetos, como los fondos, emplean bloques de código, aunque algunos bloques difieran.

En un proyecto de Scratch mediano o grande, la mayor parte del código la tendrán los objetos.

Bloques de código

Existen distintos tipos de bloques de código y están clasificados por su funcionalidad en una categoría y con un color distinto.

Estas categorías son:

Movimiento: Este código permite mover, fijar, girar, apuntar al personaje según el movimiento o acción que se le quiera dar, etc.

Apariencia: La categoría de apariencia permite controlar la apariencia del objeto/personaje, escenario, agregar viñetas para indicar que el personaje esta diciendo o pensando algo, permite también cambiar de tamaño del personaje, mostrarlo, esconderlo, disfraz (pueden ser expresiones o apariencia del personaje), efectos, etc.

Sonido: El código permite agregar y controlar el sonido de los personajes y del escenario, controlar el volumen y añadir efectos. 

Eventos: Este código sirve para ejecutar secuencias de comandos y establecer un orden a los bloques de código.

Control: Administra las acciones de los personajes, especifica cuántas veces se repite una acción o cuánto debe dura.

Sensores: Podemos realizar condiciones al haber contacto con el ratón, con un borde de la pantalla, colores concretos o con otros personajes.

Operadores: Con esta categoría podemos realizar cálculos numéricos, entre variables o valores de bloques que ya existen.

Variables: Estructuras de datos referenciadas por un nombre que pueden cambiar de contenido durante la ejecución de un programa.

Mis bloques: Podemos crear nuestros propios bloques de códigos (script), personalizados.

Existen además otros bloques especiales que podemos incluir para proyectos más avanzados. Puedes observan que debajo de "Mis bloques" aparece el icono de acceso para añadir estos otros bloques.

Eventos

De todos los bloques de código anteriores, el más importante e imprescindible son los Eventos, puesto que son estos bloques los que inician cualquier otra acción de otro tipo de bloques.
